[技能檢定]題組一 步驟5 製作後台網站標題管理功能

發表於 2021-09-23 更新於 2022-02-07 分類於 網頁設計乙級 ， 題組一

除了 total及bottom，其他七項後台的功能版面都很像，我們先以網站標題管理這個功能來做示範，先完成這個功能後，相同的程式碼可以快速套用到其它功能去；

我們需要先把素材提供的html整理一下，以符合我們解題的需要，另外，素材並沒有附上新增資料及更新圖片的表單格式，因此這部份的HTML碼要我們自己來撰寫；

另外，觀察題目所給的後台各項功能參考畫面，可以發現每個功能的結構都差不多，包括功能的標題文字，新增按鈕的文字及列表表格的文字等，因此我們可以利用原本宣告的DB類別，把各功能的文字都加進去，讓物件本身就具有提供文字資訊的功能。

再者，我們也發現在本題組中每一個功能的畫面都只會關聯到單一資料表，不像首頁那樣一個畫面中同時會有多個資料表的內容需要顯示；因此，我們可以利用PHP程式的特性，在後台載入(include)每個功能的PHP程式內容時，我們把原本分開宣告的不同資料表物件都轉成同一個變數來使用；簡單來說，當首頁在顯示時，使用的是以各資料表名宣告的物件，如$Title、$News….etc，但是在後台頁面時，使用的是統一的變數，如$DB；我們利用base.php會在各頁面被include的特性，依據每個頁面可以透過網址拿到的參數來決定這個$DB的變數會是代表那資料表：

做好以上的前置作業後，我們現在知道每個頁面都會同時有各個資料表名的物件及一個代表當前頁面的$DB變數可以讓我們視需求來選用適合的變數做運用。

接著我們開始正式的進行網站標題管理功能的製作：

  1. 先移除 back.php 中的 iframe 及修改 <form> 標籤的內容

  2. 依照功能的要求修改列表欄位的HTML碼

  3. 建立一個資料表名專用的變數($DB)來取代手動變更資料表名

  4. 在適當的位置置入相應的文字

  5. 將彈出視窗的js函式 op 套用到更新圖片按鈕中

  6. 建立一個 modal 資料夾來存放使用ajax載入的彈出視窗內容

  1. 指定新增資料/更新圖片/編輯資料三個不同功能對應的api檔案及路徑名，如果需要帶入參數的也需一併填寫。

    • 需注意功能中是否需要判斷檔案上傳的動作
    • 需注意資料表名及函式的參數引用是否適合
    • 和檔案相關的操作要注意路徑或是檔案覆蓋的問題
    • 瀏灠器不會主動更新同檔名的圖片，因此在更新圖片的功能中，我們採用更新資料表內容的方式來強迫瀏灠器去更新圖片
    • 確認 ./img/ 目錄存在，上傳的檔案才有地方放

  2. api/add.php 專門用來新增資料的程式

  3. 建立資料列表程式碼，這邊要注意的是在顯示和刪除這兩個功能，由於radio和checkbox的特性，我們會視功能要求來進行不同的設定。

  4. 做到這邊可以發現不論是modal的檔案名稱或是檔案間的傳值，我們會大量的重覆使用到資料表名來呈現，而我們在DB這個物件中原本就有設計可以使用$DB->table來顯示資料表名稱，因此我們可以把後台功能的程式碼中會使用到資料表名稱的地方都使用$DB->table來替換，只要維持這樣的規範，將可以大量的節省我們複製到其他功能時需要修改的時間。

  5. api/edit.php 專門用來更新資料的程式

  6. api/upload.php 專門用來更新上傳圖片的程式

思考點:
1. 實際解題時如果一個功能一個功能照順序做，那幾乎不太可能在時間完成，因此這邊只是示範單一功能的完整開發過程；
2. 實際應檢時，會同時考量多個類似功能的情形，並在完成一個模組後，快速複製套用；比如我們延用網址參數 do 來當成 table 的變數，應用在功能中的各個需要用到資料表名的地方，在完成標題管理的HTML程式碼後，就可以把修改好的HTML碼快速複製到類似的ad,mvim,image,news,admin,menu去，
3. 修改欄位到符合題目要求，這樣就可以快速完成前端頁面的HTML碼處理，然後再接著寫API，API的撰寫也會考量是否可以同時適用多個功能。
